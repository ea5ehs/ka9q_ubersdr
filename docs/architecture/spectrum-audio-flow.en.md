## 1. Overview

UberSDR handles two independent real-time data flows:

- Audio stream (RTP → WebSocket → client)
- Spectrum stream (STATUS → bins → WebSocket → client)

Both originate from radiod.

---

## 2. Audio Flow

### Source

Audio is generated by radiod and sent as RTP multicast.

### Backend

File: audio.go

- AudioReceiver.Start()
- receiveLoop()
- routeAudio(...)

Responsibilities:
- receive RTP
- extract SSRC
- route to session

---

### Session Routing

File: session.go

- map SSRC to session
- push to AudioChan

---

### Streaming

File: websocket.go

Function: streamAudio(...)

- read AudioChan
- encode (Opus / PCM)
- send via WebSocket

---

### Client

- websocket-manager.js
- app.js

- decode audio
- play via AudioContext

---

## 3. Spectrum Flow

### Source

Spectrum is generated by radiod (FFT).

UberSDR does not compute FFT.

---

### Backend Polling

File: user_spectrum.go

- pollLoop()
- sendPoll()

---

### Parsing

Function: parseStatusPacket(...)

Extract:
- frequency
- bin count
- bin bandwidth
- FFT bins

---

### Distribution

Function: distributeSpectrum(...)

- match SSRC
- push to SpectrumChan

---

### Streaming

File: user_spectrum_websocket.go

Function: streamSpectrum(...)

- send JSON or binary frames

---

## 4. Geometry

totalBandwidth = binCount * binBandwidth

low = centerFreq - totalBandwidth / 2  
high = centerFreq + totalBandwidth / 2

---

## 5. Summary

Audio:
radiod → RTP → backend → session → WebSocket → client

Spectrum:
radiod → STATUS → backend → session → WebSocket → client

---

## 6. Notes

- Audio and spectrum are independent
- SSRC links data to sessions
- FFT comes from radiod
